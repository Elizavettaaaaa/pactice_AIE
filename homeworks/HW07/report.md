# HW07 – Report

## 1. Datasets

### 1.1 Dataset A
- **Файл:** `S07-hw-dataset-01.csv`
- **Размер:** 12000 строк, 8 признаков.
- **Признаки:** Числовые, требуют масштабирования.
- **Особенности:** Глобулярные кластеры, вытянутые из-за разного масштаба.

### 1.2 Dataset B
- **Файл:** `S07-hw-dataset-02.csv`
- **Размер:** 8000 строк, 3 признака.
- **Признаки:** Числовые.
- **Особенности:** Нелинейная структура (вложенные формы), наличие шума.

### 1.3 Dataset C
- **Файл:** `S07-hw-dataset-03.csv`
- **Размер:** 15000 строк, 4 признака.
- **Признаки:** Числовые.
- **Особенности:** Кластеры разной плотности.

## 2. Protocol

- **Препроцессинг:** `StandardScaler` (обязательно) + `SimpleImputer`.
- **Поиск гиперпараметров:** GridSearch по сетке параметров (k для KMeans, eps для DBSCAN).
- **Метрики:** Silhouette, Davies-Bouldin, Calinski-Harabasz.
- **Визуализация:** PCA (2D).

## 3. Models

Сравнивались:
1. **KMeans:** (baseline) с перебором `k`.
2. **DBSCAN:** плотностный метод, перебор `eps`.
3. **AgglomerativeClustering:** иерархический метод (ward linkage).

## 4. Results

### 4.1 Dataset A
- **Лучший метод:** KMeans.
- **Метрики:** Silhouette > 0.5.
- **Комментарий:** KMeans отлично разделил глобулярные кластеры.

### 4.2 Dataset B
- **Лучший метод:** DBSCAN.
- **Метрики:** Silhouette ~ 0.33.
- **Комментарий:** DBSCAN справился с нелинейностью и шумом, где KMeans провалился.

### 4.3 Dataset C
- **Лучший метод:** KMeans (или Agglomerative).
- **Метрики:** Silhouette ~ 0.32.
- **Комментарий:** Несмотря на разную плотность, центроидные методы показали стабильный результат.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)
- KMeans быстр и надежен на простых формах, но не работает на "дугах" (Dataset B).
- DBSCAN идеален для очистки от шума, но сложен в настройке.
- Scaling критически важен для всех методов.

### 5.2 Устойчивость (обязательно для одного датасета)
Проверка на **Dataset A**:
- Запущено 5 раз KMeans с разными `random_state`.
- **Результат:** Метрика Silhouette идентична во всех запусках (0.3968).
- **Вывод:** Алгоритм устойчив.

### 5.3 Интерпретация кластеров
Кластеры интерпретировались через визуализацию PCA. На Dataset A видно четкое разделение на две группы по значениям признаков.

## 6. Conclusion
1. Препроцессинг обязателен.
2. Нет универсального алгоритма (KMeans для простых форм, DBSCAN для сложных).
3. Визуализация (PCA) необходима для подтверждения метрик.
